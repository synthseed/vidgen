# core-beliefs
Owner: platform
Status: active
Last Reviewed: 2026-02-23

## Belief 1: Repository Knowledge Is the Source of Truth
Agent output quality depends on clear in-repo knowledge. Decisions that are not in markdown
are treated as non-binding.

## Belief 2: AGENTS.md Is a Map, Not a Dump
`AGENTS.md` should direct agents to the right context quickly. Detailed guidance belongs in
focused docs under `docs/`.

## Belief 3: Plans Before Non-Trivial Code
Complex changes require an execution plan with checkpoints and validation steps.

## Belief 4: Mechanical Enforcement Beats Good Intentions
Use CI checks, templates, and linting to enforce documentation and review standards.

## Belief 5: Continuous Doc Gardening
Docs must be refreshed continuously to prevent entropy.
