Owner: platform

This repo adopts the following ideas from OpenAI's harness-engineering post:
- Keep root `AGENTS.md` short and use it as a context map.
- Store detailed repository knowledge in a structured `docs/` tree.
- Require requirements audits and doc updates in every substantial PR.
- Treat architecture and style constraints as explicit written policy.
- Run recurring doc-gardening to fight entropy.

- https://openai.com/index/harness-engineering/
